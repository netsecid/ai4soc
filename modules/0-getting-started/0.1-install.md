# 0.1 Install Your AI Agent

Before anything else, you need an AI agent running on your machine.

This lesson walks you through all three supported tiers. Pick the one that matches
your situation and follow only that section. You do not need to install more than one.

---

## Which Tier Should I Choose?

```
Do you have a Claude Pro or Max subscription?
├── Yes → Use Tier 2 (Claude Code) — best experience, go there directly
└── No  →
        Do you have a credit card for pay-per-use cloud services?
        ├── Yes → Tier 1 (OpenRouter) — flexible, no monthly commitment
        └── No  → Tier 0 (Ollama) — completely free, runs locally on your machine
```

All three tiers work for this course. You can always switch later.

## 🟢 Tier 0 — Ollama (Free, Local, No Account Needed)

Ollama runs AI models entirely on your machine. No internet connection required
after the initial download. No data leaves your computer. No account needed.

**Minimum requirements:**
- 8 GB RAM (for the smallest model)
- 16 GB RAM recommended (for the best local model)
- macOS, Windows, or Linux
- ~5–10 GB free disk space per model

### Step 1 — Install Ollama

Go to [ollama.com/download](https://ollama.com/download) and download the installer
for your operating system. Run the installer and follow the prompts.

Verify the installation:
```bash
ollama --version
```

You should see a version number. If you see "command not found", restart your
terminal and try again.

### Step 2 — Pull a Model

Choose based on your available RAM:

```bash
# 16 GB RAM or more — best quality for SOC tasks
ollama pull qwen2.5-coder:14b

# 8 GB RAM — good balance of quality and speed
ollama pull deepseek-r1:8b

# 4 GB RAM — lightweight, good for getting started
ollama pull llama3.2:3b
```

This downloads the model to your machine. It may take several minutes depending on
your internet speed. Each model is 4–10 GB.

> **Model note:** The AI landscape evolves rapidly. The models above are reference
> points at the time of writing. Check [ollama.com/library](https://ollama.com/library)
> for newer or better models — look for ones tagged `code` or `instruct` for best
> results on SOC tasks.

### Step 3 — Test Your Model

```bash
ollama run qwen2.5-coder:14b
```

Type a quick test:
```
>>> Summarize what a SOC analyst does in 2 sentences.
```

If you get a coherent response, your model is working. Type `/bye` to exit.

### Step 4 — Connect to ai4soc

Ollama does not have a built-in agentic interface like Claude Code. You have two options:

**Option A — Open WebUI (recommended for Tier 0)**
Open WebUI gives you a chat interface that connects to your local Ollama models.

```bash
# Requires Docker — see docs/tips/docker-basics.md if you need help
docker run -d -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  ghcr.io/open-webui/open-webui:main
```

Then open [http://localhost:3000](http://localhost:3000) in your browser.

**Option B — Terminal with a system prompt**
Use the `CLAUDE.md` content as your system prompt directly in Ollama:

```bash
ollama run qwen2.5-coder:14b --system "$(cat CLAUDE.md)"
```

Run this from inside the `ai4soc/` directory.

> **Limitation:** Tier 0 does not support parallel agents or native file operations
> the way Claude Code does. Modules that use these features will note when a Tier 0
> workaround is available.

## 🟡 Tier 1 — OpenRouter (Pay-per-Use, Cloud)

OpenRouter gives you access to many AI models — including Claude, GPT-4o, Gemini,
and Mistral — without a monthly subscription. You pay only for what you use.
Typical cost for a full module: under $0.50 USD.

**Requirements:**
- An OpenRouter account ([openrouter.ai](https://openrouter.ai))
- A credit card for billing
- Internet connection

### Step 1 — Create an Account

Go to [openrouter.ai](https://openrouter.ai) and sign up. Add a small amount of
credit to your account ($5 is more than enough to complete this course).

### Step 2 — Get Your API Key

In your OpenRouter dashboard, go to **Keys** and create a new API key. Copy it
somewhere safe — you will not be able to see it again.

### Step 3 — Install an Interface

OpenRouter works with any OpenAI-compatible client. We recommend **Aider** for
agentic SOC workflows:

```bash
pip install aider-chat
```

Or use **Continue** (VS Code extension) for a more visual experience. See
[`setup/openrouter.md`](../../setup/openrouter.md) for the full configuration guide.

### Step 4 — Configure and Test

```bash
export OPENROUTER_API_KEY=your-key-here

aider --openai-api-base https://openrouter.ai/api/v1 \
      --openai-api-key $OPENROUTER_API_KEY \
      --model anthropic/claude-sonnet-4-5
```

> ⚠️ Cloud service — always sanitize any real data before use.
> See the sanitization warning in `CLAUDE.md` and `docs/sanitization-guide.md`.

## 🔵 Tier 2 — Claude Code (Recommended)

Claude Code is a terminal-based AI agent built by Anthropic. It has native file
operations, parallel agent support, slash commands, and persistent project memory
via `CLAUDE.md`. This is what ai4soc was built and tested on.

**Requirements:**
- Claude Pro or Max subscription ([claude.ai](https://claude.ai))
- Node.js 18 or higher
- macOS, Windows (via WSL), or Linux
- Terminal access

### Step 1 — Check Node.js

```bash
node --version
```

You need version 18 or higher. If you see a lower version or "command not found":

```bash
# macOS (with Homebrew)
brew install node

# Windows — download from nodejs.org
# Linux (Ubuntu/Debian)
sudo apt update && sudo apt install nodejs npm
```

### Step 2 — Install Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

Verify:
```bash
claude --version
```

### Step 3 — Authenticate

```bash
claude
```

On first run, Claude Code will open a browser window for authentication. Sign in
with your Anthropic account. Once authenticated, return to the terminal.

### Step 4 — Test the Installation

From any directory, run:
```bash
claude
```

Type a quick test:
```
What does a SOC analyst do?
```

If you get a response, you are ready. Type `exit` or press `Ctrl+C` to quit.

> **Note:** Claude Code requires an active Claude Pro or Max subscription. If you
> do not have one, use Tier 0 or Tier 1 to start and upgrade when ready.

## Troubleshooting

**"command not found" after installing**
Close and reopen your terminal. The PATH update only takes effect in a new session.

**Ollama model download is very slow**
This is normal — models are 4–10 GB. Leave it running and come back. Downloads resume
if interrupted.

**Claude Code authentication fails**
Make sure you have an active Claude Pro or Max subscription. Free accounts do not
include Claude Code access.

**Out of memory errors with Ollama**
Switch to a smaller model. If `qwen2.5-coder:14b` is too large, try `deepseek-r1:8b`
or `llama3.2:3b`.

**Windows users**
Claude Code works best on Windows via WSL (Windows Subsystem for Linux). If you
encounter issues running Claude Code natively on Windows, see
[`setup/windows-wsl.md`](../../setup/windows-wsl.md).
