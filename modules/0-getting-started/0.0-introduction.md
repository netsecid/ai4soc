# 0.0 Introduction & Philosophy

Welcome to ai4soc.

This course teaches you how to work with AI coding agents as a security analyst. Not
as a developer. Not as a data scientist. As someone who triages alerts, hunts threats,
writes detection rules, and responds to incidents — and wants to do all of that faster
and more effectively.

---

## Why This Course Exists

The SOC is one of the most repetitive, high-stakes, and mentally demanding environments
in all of technology. Analysts stare at the same alert patterns, write the same
enrichment queries, copy-paste the same IOCs into the same lookup tools, and draft
the same incident ticket structure — day after day.

AI coding agents change this. Not by replacing the analyst, but by handling the
mechanical parts of the work so the analyst can focus on what actually requires
human judgment: pattern recognition, context, intuition, and decision-making under
uncertainty.

The problem is that most AI training material is built for developers and product
managers. Security analysts are left to figure it out themselves — translating generic
AI workflows into something that makes sense for a SOC environment.

ai4soc closes that gap.

---

## What Makes This Different

**You learn by doing real SOC work.**

Every module uses realistic lab data — alerts, logs, IOC lists, CTI reports — modeled
after a fictional Indonesian fintech company called PT Nusantara Digital. The scenarios
are designed to feel like actual incidents, not toy examples.

**You work inside the agent, not around it.**

This is not a course about prompting ChatGPT. You will install an AI coding agent,
give it persistent context about your SOC environment, and use it the way a real
analyst would — with files, slash commands, parallel tasks, and custom roles.

**It works with what you have.**

You do not need a Claude Code subscription to start. Every module supports three tiers:
fully local via Ollama, pay-per-use via OpenRouter, and full agentic mode via Claude
Code. Start where you are, upgrade when it makes sense.

**It was built by practitioners, for practitioners.**

ai4soc is maintained by BlueTeam.ID, an Indonesian cybersecurity community. The
workflows, templates, and scenarios reflect real SOC environments — not academic
exercises.

---

## The Philosophy

Three principles guide how this course is designed:

**1. Augment, don't automate.**
The goal is not to remove the analyst from the loop. The goal is to make the analyst
faster, sharper, and less burdened by mechanical work. Every workflow in this course
keeps the analyst in control and in judgment.

**2. Learn by producing real artifacts.**
Every module ends with something you created: a triage summary, a Sigma rule, an IR
timeline, a CTI report. If you complete this course, you will have a folder of
security artifacts that demonstrate real skill — not just quiz scores.

**3. Zero to productive, not zero to expert.**
This course does not try to make you a master of every tool. It tries to get you
productive with AI-augmented workflows as fast as possible. Deep dives and advanced
topics are signposted along the way — you decide how far to go.

## What You Will Build

By the end of Module 1 alone, you will be able to:

- Give an AI agent persistent context about your SOC environment
- Read log files and alert exports directly inside the agent
- Triage a SIEM alert and produce a structured summary in minutes
- Enrich an IOC list against public threat intel sources
- Run multiple analysis tasks in parallel
- Switch between analyst roles — triage, response, detection, reporting — fluidly

By the end of the full course, you will have a complete AI-augmented SOC workflow
that you can adapt to your real environment.

## How the Course Works

Each lesson is a markdown file you work through inside your AI agent. The agent reads
the lesson, understands the lab context from `CLAUDE.md`, and guides you through the
exercises interactively.

You use slash commands to navigate:

```
/start          Begin from Module 0
/check          See your current progress
/restart        Reset and start over
```

Lab files are in the `lab/` directory. Everything you produce goes into `outputs/`.

The course is self-paced. There are no deadlines, no accounts to create, and no data
sent anywhere you do not control.

## A Note on AI Models

This course was developed and fully tested using **Claude Code**. All modules are
verified to work with Claude Code and that is what we recommend for the best
experience.

Modules also include instructions for Tier 0 (Ollama, fully local) and Tier 1
(OpenRouter, pay-per-use). These are supported on a best-effort basis. The AI
landscape moves fast — if you find a model that works better than what we recommend,
open an issue and tell us. Community testing makes this course better for everyone.

## Before You Continue

Two things to do before moving to the next lesson:

**Read the disclaimers in `CLAUDE.md`.**
Everything in this lab is synthetic and safe. But if you ever bring in real data from
your own environment, the sanitization guidance matters. Read it once now so it is
not an afterthought later.

**Set your expectations.**
This course will make you faster. It will not make security easy. Judgment, context,
and experience still matter. The AI is a tool — a powerful one — but the analyst is
still the professional.
