# 0.2 Clone the Repo & Explore the Lab

You have an AI agent installed. Now let's get the course files and understand
what you are working with.

---

## Step 1 — Clone the Repository

Open your terminal and run:

```bash
git clone https://github.com/blueteamid/ai4soc.git
cd ai4soc
```

If you do not have Git installed:

```bash
# macOS
brew install git

# Ubuntu / Debian
sudo apt update && sudo apt install git

# Windows — download from git-scm.com
```

Verify the clone worked:

```bash
ls
```

You should see: `CLAUDE.md`, `README.md`, `lab/`, `modules/`, `setup/`, `connectors/`, `outputs/`

---

## Step 2 — Understand the Structure

Here is what each folder contains and why it matters:

```
ai4soc/
│
├── CLAUDE.md              The brain of your AI agent for this course.
│                          Every time you start a session inside this folder,
│                          the agent reads this file and understands your SOC
│                          context, available tools, workflows, and commands.
│
├── lab/                   All your practice material lives here.
│   ├── alerts/            SIEM alert exports — JSON and CSV format
│   ├── logs/              Raw logs — Windows Event, Linux syslog, CloudTrail
│   ├── ioc/               IOC lists — IPs, hashes, domains, URLs
│   ├── reports/           CTI reports for analysis exercises
│   └── scenarios/         Full threat campaign simulations
│
├── modules/               The course content — you are reading this now.
│   ├── 0-getting-started/
│   ├── 1-fundamentals/
│   ├── 2-soc-workflows/
│   ├── 3-automation/
│   └── 4-advanced/
│
├── setup/                 Installation guides for each AI tier.
│
├── connectors/            Integration guides for connecting to real SIEMs.
│   └── wazuh/
│
└── outputs/               Everything you produce goes here.
                           Create a dated subfolder for each session:
                           outputs/2025-01-15/
```

**Two rules to remember:**
- `lab/` is read-only — treat it as evidence, never modify it
- `outputs/` is where you write everything — triage reports, rules, scripts, notes

---

## Step 3 — Open the Lab in Your Agent

This is where the course comes alive. Navigate into the `ai4soc` folder and start
your AI agent. The agent will automatically read `CLAUDE.md` and load your SOC context.

**Claude Code:**
```bash
cd ai4soc
claude
```

You should see Claude Code start up. It will read `CLAUDE.md` automatically because
it looks for that file in the current directory. Your agent now knows it is operating
in a SOC environment.

**Ollama (Open WebUI):**
Start your session and paste the contents of `CLAUDE.md` as your system prompt.
See [`setup/ollama.md`](../../setup/ollama.md) for the exact steps.

**OpenRouter / Aider:**
```bash
cd ai4soc
aider --openai-api-base https://openrouter.ai/api/v1 \
      --openai-api-key $OPENROUTER_API_KEY \
      --model anthropic/claude-sonnet-4-5 \
      --read CLAUDE.md
```

## Step 5 — Meet PT Nusantara Digital

All lab scenarios are set in the fictional company your agent knows from `CLAUDE.md`:

**PT Nusantara Digital** is a fintech company based in Jakarta, Indonesia.
About 500 employees. They process payments, hold customer financial data, and
operate under PCI-DSS and OJK compliance requirements.

Their security stack:
- Wazuh for SIEM
- CrowdStrike Falcon for endpoint detection
- AWS as primary cloud, GCP secondary
- Zeek and Suricata for network visibility

Their threat profile: financially motivated actors, ransomware groups, BEC campaigns,
credential theft operations. The kind of threats that hit real Indonesian fintechs.

When you work through exercises, think of yourself as the SOC analyst on duty for
this company. The alerts are real (to the scenario). The decisions matter (to the
learning). The artifacts you produce should look like something you would actually
file in a real incident.

## Step 6 — Create Your First Output Folder

Before your first exercise, create today's output directory:

```bash
mkdir -p outputs/$(date +%Y-%m-%d)
```

Or ask your agent to do it:
```
Create today's output folder in the outputs directory.
```

All files you produce during this course go here, organized by date.

## What Just Happened

When you started your agent inside the `ai4soc/` folder, it loaded `CLAUDE.md`
and gained persistent context about:

- The fictional organization it is helping you protect
- The security stack in the lab environment
- The roles you play as an analyst (triage, IR, CTI, detection, reporting)
- The output templates to use for each type of artifact
- The slash commands available for navigating workflows

This is called **project memory** — and it is one of the most powerful features
of AI coding agents. Unlike a chat session that forgets everything when you close
the tab, your agent now has a consistent, reloadable SOC identity every time you
start it inside this folder.

In Module 1.6, you will learn how to customize `CLAUDE.md` for your own
environment. For now, the defaults are set up for the lab.
