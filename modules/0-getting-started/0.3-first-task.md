# 0.3 Your First SOC Task in 5 Minutes

This is where it gets real.

You have an AI agent running with SOC context loaded. You have lab data to work with.
In the next five minutes, you are going to triage a security alert the way an
AI-augmented analyst does it — and produce a structured output you could actually file.

---

## The Scenario

It is 09:15 WIB on a Tuesday morning. You are the SOC analyst on duty at
PT Nusantara Digital.

Wazuh has fired an alert. A Windows workstation belonging to a finance department
employee has generated multiple failed authentication events followed by a successful
login from an unusual IP address. The alert landed in your queue three minutes ago.

Your job: determine whether this is a real threat or a false positive, and document
your findings.

---

## Step 1 — Load the Alert

From inside your agent session, run:

```
/triage lab/alerts/scenario-01-alert-bruteforce.json
```

Or if you are not using slash commands, paste this into your agent:

```
I need to triage a security alert. Please read the file
lab/alerts/scenario-01-alert-bruteforce.json and help me analyze it.
Start by summarizing what the alert is telling us.
```

Your agent will read the file and give you an initial summary of what it contains.

---

## Step 2 — Ask the Right Questions

A good analyst does not just read the alert — they investigate. Use your agent to
work through the key questions:

**Enrich the source IP:**
```
Enrich the source IP from this alert. Check what we know about it —
reputation, geolocation, ASN. Use AbuseIPDB and VirusTotal references.
```

**Check the timeline:**
```
Build a timeline of events from this alert. When did the failed attempts start,
when did the successful login happen, and how long was the gap?
```

**Assess the asset:**
```
What do we know about the affected workstation? Is it a high-value asset
based on the organizational context in CLAUDE.md?
```

**Look for related activity:**
```
Based on what you see in this alert, what other log sources or alert types
should we check to understand the full scope?
```

Take your time with each question. The agent will reason through each one using the
lab data and the SOC context it loaded from `CLAUDE.md`.

---

## Step 3 — Make a Verdict

After you have gathered the evidence, it is time to make a decision. Ask your agent:

```
Based on everything we have analyzed, what is your assessment?
Is this a True Positive, False Positive, or does it need more investigation?
Give me your reasoning.
```

The agent will give you a verdict with reasoning. Read it critically. Do you agree?
Is there anything the agent missed? The verdict belongs to you — the agent is a
thinking partner, not the decision-maker.

---

## Step 4 — Produce the Output

Now write it up. Ask your agent:

```
Using the Alert Triage Summary template from CLAUDE.md, produce a complete
triage report for this alert based on our analysis. Save it to
outputs/[today's date]/triage-scenario-01.md
```

The agent will fill in the template with everything you investigated and save the
file to your outputs folder.

Open the file. Read it. This is a real artifact — the kind of thing you would
attach to a Jira ticket or send to a team lead. If anything looks wrong or
incomplete, tell the agent and it will fix it.

---

## What You Just Did

Walk through what happened there:

**Without AI:** You would have manually opened the alert, copied the IP into
VirusTotal, switched to another tab for AbuseIPDB, built the timeline by hand in
a text editor, and written the triage report from scratch. Minimum 15–20 minutes
for a straightforward alert.

**With AI:** You described what you needed, the agent read the file, reasoned
through the evidence, and produced a structured output. Under 5 minutes. And the
quality of the output is consistent — it follows the same template every time,
with the same sections, the same format.

Now multiply that across 50 alerts in a shift.

This is the value of AI-augmented SOC work — not magic, just leverage.

## Things to Try

Before moving to Module 1, try a few variations to get comfortable with the workflow:

**Change the verdict:**
```
Actually, I think this might be a False Positive because the user
travels frequently. Update the triage report with this additional context.
```

**Ask for a different format:**
```
Generate an alert notification for this incident in Slack message format
using the template from CLAUDE.md. Keep it short and non-technical.
```

**Challenge the reasoning:**
```
What would change your assessment from True Positive to False Positive?
What additional evidence would you need?
```

**Try a second alert:**
```
/triage lab/alerts/scenario-02-alert-phishing.json
```

## A Word on Accuracy

Your agent will sometimes get things wrong. It may misread a timestamp, miss a
field in the log data, or make an assumption that does not hold up. This is normal
and expected.

Your job as the analyst is to stay engaged, question the output, and correct it
when needed. The agent is fast but not infallible. You are the professional with
the full context — use it.

When the agent makes a mistake, correct it directly:
```
That IP geolocation is wrong — the alert shows the source is 103.x.x.x
which is Indonesian IP space, not Eastern Europe. Revise the assessment.
```

The agent will update its reasoning and the output accordingly. This back-and-forth
is the real workflow. It is not about getting a perfect answer in one shot — it is
about iterating toward the right answer faster than you could alone.
